# KNOWLEDGE - ダム前後の縦断描画・収束監査

## 基線と原因

- 前サイクルの詳細知識は`.spec/archive/2026-07-16-pre-dam-profile-rendering/KNOWLEDGE.md`へ保存した。開始commitは`40cbb50`、自己試験68/68、標準1200秒24,001 step・243 frame、HTML約310KBだった。
- 参照河道は0～300mを10m間隔の31セル中心で表すため、有限体積の計算面は5、15、…、295mにある。入力ダム100m・200mは同距離候補の上流側を採る規則により、実際には95m・195mの面へ割り当てられる。
- Workerは95m・195mで流束を制限していた一方、旧描画は黄色線を入力値100m・200mへ置き、水面・移動床の左右セル中心値を一本の斜線で結んでいた。画像の三角スパイクは、物理的な左右状態差、5mの表示ずれ、セル中心間の直線補間が重なったものだった。
- したがって、平滑化や表示専用クランプは使わず、計算面と左右セル平均値をそのまま可視化する方針を採った。

## 実装した境界とコンポーネント

- Workerとmainの双方へ純粋な`resolveStructureFaces`と単数wrapperを実装した。セル列は1回検証し、各要求位置を二分探索するため計算量は`O(N + D log N)`である。
- 戻り値は`face`、`leftIndex`、`rightIndex`、`faceX`、`offset`。等距離は32 ULP以内なら上流側を採る。許容幅は局所距離と局所面間隔だけから作り、絶対座標値や単位依存の絶対下限を使わない。
- 極大座標`[1e16, 1e16+4, 1e16+8]`、微小座標`[0, 1e-16, 2e-16]`、小数格子のULP tie、非一様格子、完全一致、表現不能なface中点、無効入力をWorker/main両方で回帰した。
- Workerは受信した`face`/`faceX`を信用せず、検証済み`cells1d`と`structure.x`から独立再計算する。mainも検証済みケースとframeから表示値を再導出し、Workerの`frame.x`がケース座標と完全一致しない応答を拒否する。
- `buildFaceAwareProfilePath`は、水面と移動床だけに`左中心 -> (faceX, 左値) -> (faceX, 右値) -> 右中心`を挿入する。固定床は連続線のまま、0件・1件・隣接複数ダム、入力配列不変を試験した。
- ダム線は`faceX`へ描き、基部を`max(frame.bed[left], frame.bed[right])`、天端を基部＋高さとした。天端をy-rangeへ含め、DOM/Canvasに「段差はダム計算面の左右セル値」を明示した。
- UIは要求位置を変更せず、参照ケースで`入力 100 m / 計算面 95 m / 差 -5 m`、`入力 200 m / 計算面 195 m / 差 -5 m`と表示する。CSV schema 1、case hash項目、Worker protocol 1、frame履歴は変更していない。

## 31・61・121点の格子収束監査

- 確率軌道を空間離散誤差へ混ぜないため、監査用派生ケースだけ全ダムの`blockageProbability=0`とし、Seedを固定した。31・61・121点を1200秒まで実行し、900秒は前後frameから時間線形補間した。
- 実効ダム面は31点で95/195m、61点で97.5/197.5m、121点で98.75/198.75mとなり、要求位置への誤差は5→2.5→1.25mへ半減した。
- 0～300mの同じ物理領域で、各要求ダム位置の前後10mを全格子共通で除外し、セル平均を区分一定値としてcontrol-volume重なり積分した。除外域だけを変更してもL1とscaleが変わらないこと、標高を平行移動しても水面rangeが変わらないことを単体試験した。

### 実行健全性

- 31点: 24,001 step、243 frame、水収支誤差`5.644e-5`、土砂収支誤差`1.097e-6`、最小水深`0.0705m`、最大CFL`0.3543`。
- 61点: 29,755 step、243 frame、水収支誤差`3.982e-5`、土砂収支誤差`2.082e-7`、最小水深`0.0511m`、最大CFL`0.42`。
- 121点: 46,381 step、243 frame、水収支誤差`3.604e-5`、土砂収支誤差`4.050e-8`、最小水深`0.0363m`、最大CFL`0.42`。
- 全値有限、非負水深、設定CFL以下、濃度範囲内を満たした。開境界の水・土砂誤差は既存範囲`1e-4`/`2e-6`内で細分化により改善したが、これは回帰監視であり閉領域保存精度の証拠ではない。閉領域の既存保存試験は`1e-8`以下を維持した。

### 自己収束値

- 900秒のL1は、水面`0.8745 -> 0.3259m`で次数`p=1.424`、水深`0.5829 -> 0.1775m`で`p=1.716`、河床変動`0.09241 -> 0.03653m`で`p=1.339`。
- 1200秒のL1は、水面`0.8798 -> 0.3412m`で次数`p=1.367`、水深`0.5942 -> 0.2024m`で`p=1.554`、河床変動`0.1120 -> 0.03307m`で`p=1.759`。
- すべての自己収束次数は仕様の`p>=0.7`を満たす。一方、同一支持領域で121点水深scaleに対する61～121差は900秒`29.9%`、1200秒`67.8%`で2%目安を大幅に超えた。よって「細分化で差は縮小する」は成立するが、121点解の絶対水深を格子独立とは認定しない。
- 900秒・100mダム直下流の水深は31/61/121点で`1.1366 -> 0.5743 -> 0.9663m`、対応faceXは`95 -> 97.5 -> 98.75m`で、局所traceは単調収束しなかった。ダム近傍を除外した主profileの自己収束と、移動する不連続直近の局所未収束を分離して扱う。
- この未達は描画で隠さない。ダム流束や河床更新の変更は本サイクルの対象外であり、絶対局所値が必要なら別SDDで局所格子・不連続追跡・現地校正を検討する。

## 独立監査

- 独立read-only監査タスクは`019f6979-cefb-7d90-96e2-a6253ff5b7d8`。初回指摘を受け、確率閉塞を無効化した決定論的監査、同一支持領域scale、区分一定L1、絶対座標に依存しないtie許容幅、表現不能面拒否を追加した。
