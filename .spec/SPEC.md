# SPEC - KANAko 1.44 挙動比較・設定編集改善

## 1. 基線と適用範囲

- 既存完成仕様は `.spec/archive/2026-07-16-pre-kanako-behavior-audit/SPEC.md` を基線とし、本仕様はその要件を削除せず追加する。
- 成果物、物理モデル、CSV、Workerプロトコル、信頼境界、384MiB上限、700KiB目標は従来仕様を維持する。
- KANAko 1.44はUI操作と出力比較の参照であり、実装コード、画像、非公開アルゴリズムを流用しない。
- canonical数値出力がないため、KANAkoとのピーク・到達・河床差は引き続き妥当性合格条件にしない。

## 2. 実機で確認した比較対象

- メイン画面: 新規作成、河川データ読込・保存、計算開始・停止、結果保存、ダム追加、観測点追加、流入ハイドログラフ、フィールド設定、堆積層厚一括設定。
- フィールド設定: 計算点数30～50。範囲外は明示エラーにする。
- 地点設定: 上流端からの距離を識別子として、移動床標高、固定床標高、川幅を編集する。
- 流入設定: 時系列流量曲線、2粒径の濃度、指定時刻以後の濃度変更を設定する。
- ダム設定: 不透過、スリット、格子を切替可能。スリットは幅、格子は横桁本数・柱径・柱中心間隔の専用設定を持つ。
- KANAkoでは格子型の高さ欄が無効で、専用画面の既定値は横桁5本、柱径0.5 m、柱中心間隔2.2 m。格子型からスリット型へ切り替えた直後のスリット幅は0 mだった。
- KANAkoはスリット幅欄の文字列 `1-1` を変更時に拒否せず、再表示時に1として保持した。互換対象は画面構成と操作意図に限定し、この緩い数値解釈は再現しない。
- 観測点: 追加後に全観測点のハイドログラフを同時表示し、個別画面から削除できる。
- 計算中: 設定操作を無効化し、開始ボタンを停止へ切替え、時刻・水面・河床・粒径別流量を逐次描画する。

## 3. 流入ハイドログラフ編集

- ケース設定へ `InflowEditor` を追加し、各行の時刻、流量、全粒径の濃度を表形式で編集できるようにする。
- 行の追加・削除を提供し、最低2点、時刻の狭義単調増加、`0 <= time <= duration`、非負流量、各濃度の有限・非負、濃度合計上限を既存検証層で保証する。
- 粒径数が変わらない限り濃度配列長を保持し、行追加時は隣接点を線形補間した決定論的な値を使う。
- KANAkoの「途中で濃度変更」は、汎用時系列行で表現する。隠れた専用分岐やKANAkoだけの時刻判定は追加しない。
- 流量グラフは計算前から流入曲線を描き、計算後は選択観測点の流量を重ねて区別可能な凡例を表示する。

## 4. 河道・計算点編集

- `GeometryEditor` を追加し、計算点数、選択点、移動床標高、固定床標高、川幅を画面から編集できるようにする。
- 計算点数UIは2～10,000の既存上限内とし、KANAko互換目安が30～50であることを説明する。
- 点数変更は明示的な「再標本化」操作でのみ行い、既存x範囲の端点を保存して、固定床、移動床、川幅、初期水深を線形補間する。構造物と観測点の物理座標は変更しない。
- 地点編集ではxを読取専用とし、`bed >= fixedBed`、`width > 0`、有限数を必須にする。無効入力を部分適用しない。
- 「堆積層厚を全区間へ適用」を追加し、有限・非負値について全点を `bed = fixedBed + thickness` に更新する。KANAkoの0～10mを互換目安として表示するが、既存モデル上限内の値だけを受理する。
- 点数変更、地点変更、一括適用は結果frameを破棄し、再検証後のhashへ更新する。

## 5. ダム種類と追加位置

- 不透過、スリット、格子の3種類を選択できる既存機能を維持し、種類変更を即時に正規化済みケースへ反映する。
- 共通項目は位置、高さ、流量係数、堆砂容量とする。
- スリット型だけにスリット幅を表示し、0以下の幅を持つ他形式から切替えた場合は本HTMLの新規ダム既定値1 mを設定する。これはKANAkoの0 m初期値を再現するものではなく、本HTMLのモデルで切替直後から開口を機能させるための意図的な安全既定値とする。
- 格子型だけに有効格子間隔、閉塞確率、再移動せん断を表示し、0以下の間隔を持つ他形式から切替えた場合は既存新規ダム既定値0.5mを設定する。
- 不透過型ではスリット・格子固有項目を表示しない。ただし形式を往復した際に明示入力済みの値はケース内に保持する。
- KANAkoの横桁本数・柱径・柱中心間隔と本実装の確率閉塞付き有効間隔は同一モデルではない。今回は偽の一対一換算を実装せず、モデル差をダム欄へ明示する。
- ダム追加は未使用の計算面から領域中央に最も近い位置を決定論的に選ぶ。使用可能面がない場合は追加しないで理由を表示する。
- 複数の構造物が同じ計算面へ写像される入力は、片方だけが暗黙に使われないよう検証エラーにする。

## 6. 観測点追加

- 観測点追加は既存観測点と異なる計算点を、領域中央に近い順で選ぶ。同じxへの自動追加を禁止する。
- 全計算点が使用済みの場合は追加しないで理由を表示する。
- 名称、位置、削除、表示対象選択の既存機能を維持する。
- KANAkoの全観測点同時小グラフは狭幅・軽量性と両立しないため、選択観測点を一つの高解像度グラフへ重ねる現行方式を維持し、差として明示する。

## 7. 実行中の状態整合性

- `running`、`paused`、`cancelling`、`validating`、ensemble実行、DEM取得の間はケース編集を無効化する。
- UIが操作可能に見えるだけでなく、すべての編集入口でbusy状態を再検査し、ケース、hash、frame、Worker入力の混在を防ぐ。
- 実行開始時の検証済みケースhashをrunへ紐づけ、完了・frame表示中に現在ケースhashと不整合を生じさせない。
- 停止完了またはエラー復旧後は編集を再有効化する。

## 8. 入力検証と互換性

- UI追加後も `parse -> normalize -> validate -> freeze -> dispatch` の順序を変えない。
- CSVは既存schema 1と全項目を往復し、UI編集した流入、河道、ダム、観測点のhash一致を確認する。
- 新しいUIは既存ケースデータの表現だけを編集し、独自保存形式やlocalStorageを追加しない。
- 削除操作は既存の明示ボタンに限定し、種類変更や再標本化で別オブジェクトを暗黙に削除しない。

## 9. 検証要件

- 既存57自己試験、Worker数値試験32件を全件回帰させる。
- ダム形式切替の既定値、形式固有項目、同一面拒否、空き位置選択、観測点非重複、再標本化、堆積層一括適用、流入編集の単体試験を追加する。
- 不透過・スリット・格子について、形式固有パラメータを変えたとき限流・粒径通過結果が期待方向へ変わることを直接試験する。
- 実ブラウザで、計算中編集不可、停止後編集可、流入曲線表示、全編集UI、リセット、CSV往復、自己試験、代表計算を確認する。
- console error/warning 0、重複ID 0、390px overflow 0、キーボード操作可能、外部必須resource 0、700KiB以下を確認する。
