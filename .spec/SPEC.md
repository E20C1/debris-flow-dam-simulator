# SPEC - ダム前後の縦断描画修正・格子収束監査

## 1. 基線と信頼境界

- 完成済みのKANAko比較サイクルは`.spec/archive/2026-07-16-pre-dam-profile-rendering/`を基線とする。
- 入力は引き続き`parse -> normalize -> validate -> freeze -> dispatch`を通し、Workerは構造物の`face`や`faceX`をクライアントから受け取らず、検証済み`cells1d`と`structure.x`から再計算する。
- mainは表示用の実効計算面を、現在の検証済みケースと表示frameから再導出する。CSV、DOM、Canvas上の派生値を数値計算の正としない。
- CSV schema 1、Worker protocol 1、frame buffer、case hashの対象項目は変更しない。

## 2. 計算面投影

- `resolveStructureFaces(cells, requestedXs)`と単数wrapperを純粋関数として追加し、`face`、`leftIndex`、`rightIndex`、`faceX`、`offset = faceX - requestedX`を返す。
- face候補は隣接セル中心の中点とし、距離最小の面を選ぶ。左右faceまでの局所距離差が、局所距離・face間隔を基準にした32 ULP以内なら上流側、すなわち小さいface indexを選ぶ。絶対座標値を許容幅へ使わず、単位依存の絶対下限も設けない。
- 2点未満、非有限座標、非増加xなどの無効入力では派生値を生成せず、既存検証層で拒否する。
- Workerのstate生成とmainの重複面検証・追加位置・表示は同一のtie-break仕様を使う。Worker側では独立再計算を維持する。

## 3. ダム設定UI

- `dam.x`は要求位置としてCSVとcase内に保持し、自動snapや暗黙変更をしない。
- 各ダムの詳細欄に`入力 200 m / 計算面 195 m / 差 -5 m`の形式で実効位置を表示する。差0の場合も入力と計算面を表示する。
- 実効位置表示は検証済みまたは現在の正規化可能なケースから毎回再計算し、編集・再標本化後に更新する。

## 4. 縦断描画

- 固定床は従来どおりセル中心を結ぶ連続線とする。
- 水面と移動床は、ダムが割り当てられた隣接セル間だけ`左セル中心 -> (faceX, 左値) -> (faceX, 右値) -> 右セル中心`の段差パスとする。
- ダムのないセル間は従来どおり直線補間する。複数ダムはface index順に適用し、同一faceは既存検証で拒否する。
- `buildFaceAwareProfilePath(xs, ys, resolvedFaces)`を純粋関数にし、Canvas座標へ変換する前のデータ座標命令列を返す。入力配列は変更しない。
- ダム線は`faceX`へ描画する。基部は数値モデルのcrest定義と同じ`max(frame.bed[leftIndex], frame.bed[rightIndex])`、天端は基部＋`dam.height`とする。
- Canvas内に「段差はダム計算面の左右セル値」と短く表示する。物理的不連続を隠す平滑化は追加しない。

## 5. 軽量性とコンポーネント境界

- 投影はセル列を1回検証した後に各要求位置を二分探索し、`O(N + D log N)`とする。1回の描画・UI render・Worker state生成では一括結果を再利用する。パスは小さな数値配列とし、永続キャッシュや新しいframe履歴を増やさない。
- DOMコンポーネントは既存のダム詳細レンダラーへ実効位置メタ情報を追加するだけとし、framework・画像・font・外部ライブラリを追加しない。
- 新しいWorker転送データ、localStorage、独自保存形式を追加しない。

## 6. 格子収束監査

- 参照ケースの河道を既存`resampleCells1D`で31・61・121点へ再標本化し、構造物の要求位置は100m・200mのまま保持する。空間離散誤差と確率軌道を混同しないよう、監査用派生ケースだけ全ダムの`blockageProbability=0`とする。
- 各格子の実効面は95/195m、97.5/197.5m、98.75/198.75mとなり、要求位置への誤差が5→2.5→1.25mへ半減することを確認する。
- 1200秒まで実行し、0秒、最大流入時刻900秒、1200秒を比較する。900秒を挟むframeを時間線形補間し、格子ごとの出力時刻超過を比較誤差へ混ぜない。
- 主比較は入力物理座標0～300m上で行い、両ダムの`|x - dam.x| < 10m`を全格子共通で除外する。セル平均は区分一定値としてcontrol-volume境界の共通分割上で重なり積分し、不連続を線形補間しない。
- 主指標は水面`eta`、水深`h = surface - bed`、河床変動`delta_zb = bed(t) - bed(0)`のL1とする。自己収束の合格は`p = log2(E31-61 / E61-121) >= 0.7`とし、数値floor以下ではpを省略できる。
- 格子収束と121点解の絶対精度を混同しない。同じ除外後支持領域で求めた121点characteristic scaleに対する61～121差を別の解像度適合指標として、水面・水深2%以下、河床変動5%以下を目安に報告する。scaleは水面=`max-min`、水深=`max(h)`、河床変動=`max(max-min, max(abs(delta_zb)))`とし、水面の標高datumや除外域の値で合否を変えない。この指標が未達なら自己収束を取り消さず、「121点でも格子独立な絶対値とは認定しない」と明記する。
- ダム直上流・直下流traceと捕捉量は、不連続を跨いだ値を作らず補助診断として全値を報告する。格子ごとにfaceXが異なるtraceだけで主profileを不合格にせず、局所解像度の制約として分離する。
- 必須integrity gateは全値有限、最小水深`>= -1e-10`、`maxCfl <= configuredCfl * (1 + 1e-12)`、濃度範囲内とする。開境界参照ケースの保存診断は既存基線範囲の水`<= 1e-4`、土砂`<= 2e-6`かつ細分化で改善することを回帰監視し、これを閉領域保存精度の証拠には使わない。閉領域の既存試験は`<= 1e-8`を維持する。
- 監査不合格時は描画平滑化や閾値緩和で隠さず、ダム流束・河床更新を別SDDサイクルの数値課題として切り分ける。
