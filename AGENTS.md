# Project guide line

## プロジェクトの原則

- プラン、仕様、TODO、検証記録、回答は日本語で作成する。
- 承認済みの `.spec/SPEC.md` を正とし、範囲を黙って縮小しない。
- 配布成果物は `debris_flow_dam_simulator.html` 単一ファイルとする。
- 外部フレームワーク、CDN、外部フォント、画像素材を必須依存にしない。
- KANAkoの既存コード・画像・非公開実装を流用せず、公開資料と出力比較によるクリーンルーム実装に限定する。
- クライアント入力、CSV、URL、Workerメッセージ、キャッシュ、外部タイル、計算結果を信頼境界として検証する。
- UI表示値を数値計算の正とせず、正規化済みケースとWorker内の検証済み状態を正とする。
- 無制限の格子数、ステップ数、Worker数、メモリ使用、外部タイル取得を許可しない。
- 物理式や保存則と異なるサイト固有ハック、結果合わせのマジックナンバー、エラー握り潰しを禁止する。
- 高度二相流は検証が完了するまで「実験」と明示し、標準モデルと混同しない。

## 作業順序

1. `.spec/PLAN.md` で目的を確認する。
2. `.spec/SPEC.md` で要件・合格基準・信頼境界を確認する。
3. `.spec/TODO.md` で次の未完了項目を一つ選ぶ。
4. 実装し、該当範囲の自己試験・ブラウザ試験・数値試験を行う。
5. 証拠が揃った項目だけTODOへチェックを入れる。
6. 判断・失敗・数値閾値を `.spec/KNOWLEDGE.md` に記録する。

## コンテキスト圧縮対策

- 作業開始・再開・最終報告前に、目的、制約、未完了、検証証拠を短く再確認する。
- `.spec/TODO.md` を唯一の完了チェックリストとし、チャット上の記憶だけで完了判定しない。
- `.agent/handoff/HANDOFF.md` に現在フェーズ、直前の検証、次の一手、ブロッカーを具体的に維持する。
- 恒久的な知見は `.agent/memory/MEMORY.md`、今回の技術判断は `.spec/KNOWLEDGE.md` に分離する。
- MEMORY更新前は `.agent/memory/YYYY-MM-DD.md` へ既存内容をアーカイブする。
- HANDOFF更新前は `.agent/handoff/YYYY-MM-DD-HHMM.md` へ既存内容をアーカイブする。

## 完了判定

- 「テストが落ちなかった」だけで完了としない。
- TODOの各要件について、コード、自己試験、数値誤差、ブラウザ操作、CSV往復、性能計測の直接証拠を確認する。
- 未証明、間接証拠、検証対象外は未完了のまま残す。
- 作業で起動した開発サーバーや不要なバックグラウンドプロセスは最終報告前に停止する。
